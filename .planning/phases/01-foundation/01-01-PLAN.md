    **Step 1: Create `apps/brand/app/layout.tsx`.**
    Root layout (Server Component — do NOT add "use client"). Must include:
    - `import { ThemeProvider } from "next-themes"`
    - `import { Inter } from "next/font/google"`
    - `import "./globals.css"`
    - `export const metadata` with title "Brand Kit" and description "Design system reference for the monorepo"
    - `<html lang="en" suppressHydrationWarning>` — the `suppressHydrationWarning` is REQUIRED because next-themes modifies the html element's class after hydration
    - `<body className={inter.className}>` with `<ThemeProvider attribute="class" defaultTheme="system" enableSystem>` wrapping `{children}`
    - `attribute="class"` is REQUIRED — matches the `@custom-variant dark (&:where(.dark, .dark *))` in `@infrastructure/ui/globals.css`
    - JSDoc comment on the default export function

    **Step 2: Create `apps/brand/components/theme-toggle.tsx`.**
    Client component (MUST have `"use client"` directive). Must include:
    - `import { useTheme } from "next-themes"`
    - `import { Sun, Moon } from "lucide-react"`
    - `import { useEffect, useState } from "react"`
    - The `mounted` guard pattern: `const [mounted, setMounted] = useState(false); useEffect(() => setMounted(true), []);`
    - Before mounted, return a placeholder `<div className="size-9" />` to prevent hydration mismatch
    - After mounted, render a `<button>` that calls `setTheme(theme === "dark" ? "light" : "dark")` on click
    - Button styled with: `className="flex size-9 items-center justify-center rounded-md border border-border bg-background text-foreground hover:bg-accent"`
    - `aria-label="Toggle theme"` for accessibility
    - Show `<Sun>` icon when dark (to indicate "switch to light"), `<Moon>` icon when light (to indicate "switch to dark")
    - Icons sized with `className="size-4"`
    - Named export: `export function ThemeToggle()`
    - JSDoc comment on the export
