Memory recall telemetry — close the feedback loop

Status: Approved · Date: 2026-06-02 · Branch: feat/memory-recall-telemetry · Issue: #159

1. Problem

The .woostack/memory/ store writes confidently but never measures. There is no signal on whether a note ever helped: distillation writes blind and never learns which notes earn their keep. This is the root cause of curation rot — the store cannot tell live notes from dead weight, so it grows monotonically and recall quality decays.

recall.sh already computes, every run, exactly the signal that would fix this: a per-note scope-match count (the matched tmpfile, sorted -k1,1nr). Today that signal is used to order output and then discarded. Nothing is persisted, so the store stays write-only.

2. Goal

Convert the memory store from write-only to self-correcting by persisting recall usage back onto each note, then surfacing a prune signal.
  • Stamp two tool-managed frontmatter fields on every note the recall algorithm loads: recall_count (cumulative count of recall runs that loaded it) and last_recalled (ISO date of the most recent load).
  • Teach doctor.sh to derive a dead-note warning from those fields: old + never recalled = prune candidate.
  • Document the new fields and the doctor behavior in the memory contract.
Nearly free: the data is already in hand. This is the foundation later staleness/ranking work builds on.

3. Non-goals

  • No "recalled-but-never-acted-on" detection. The issue floats it but flags it as needing a downstream act signal that does not exist yet. Out of scope; recall_count alone ships here.
  • No automatic pruning. doctor warns; a human prunes. No note is ever deleted by tooling.
  • No ranking / cap-eviction changes. Recall ordering and the RECALL_CAP budget logic are untouched. Telemetry is a side channel.
  • No change to the flat memory.md shard. It remains free-form and untracked by this tooling.
  • No config schema. Thresholds are env knobs, not config.json keys.

4. Approach

Three decisions, locked during brainstorming:

DecisionChoiceWhy
recall_count semanticsCumulative recall events (+1 per run that loads the note)The "zero recalls → dead" signal only works if the count measures times-loaded. Stamps matched, linked, and global notes uniformly — linked/global have no match-count of their own.
Write-back behaviorUnconditional, best-effortAlways stamp in place; never fail recall if the write fails. Ephemeral CI clones simply don't contribute; local/persistent checkouts accrue the signal.
Stamp on selection vs emissionAll selected notesSelection = relevance. The RECALL_CAP drop is a display budget, not a relevance verdict, so a cap-dropped note still counts as recalled.
Dead threshold90 days (WOOSTACK_DEAD_DAYS)A full quarter written but never loaded = strong prune signal; tolerant of rarely-touched subsystems.

In-place frontmatter mutation

The risky primitive. Chosen mechanism: an awk rewrite of the frontmatter block, written to a temp file and mv'd over the original (atomic). It replaces the key's line if present, else inserts before the closing --- fence, preserving the body and all other fields verbatim.

Rejected: sed -i per key (cannot cleanly insert-when-absent; -i syntax differs BSD vs GNU); full reserialize via field()/note_body() (lossy — normalizes/reorders frontmatter).

5. Components & data flow

lib.sh — new shared helpers

HelperContract
set_field <file> <key> <value>awk-rewrite the frontmatter block: update key: line if present, else insert before the closing fence. Atomic temp+mv. Preserves body and other fields. Returns non-zero on write failure without clobbering the original.
_woo_nowEcho ${WOOSTACK_NOW:-$(date +%F)} — today's ISO date, overridable for deterministic tests.
_woo_epoch <YYYY-MM-DD>ISO date → Unix epoch seconds, time-of-day zeroed for determinism. Tries GNU date -d "$d 00:00:00", falls back to BSD date -j -f "%Y-%m-%d %H:%M:%S" "$d 00:00:00". Empty/non-zero on unparseable input. Zeroing is mandatory: BSD date -j -f "%Y-%m-%d" on a bare date injects the current time-of-day (verified: differs run-to-run), which would make age math and tests nondeterministic.

set_field on malformed notes: recall selects a note with no/empty frontmatter as "global" (empty scope ⇒ is_global) and will attempt to stamp it. If set_field cannot find two --- fences it returns non-zero without writing — recall then logs stamp failed and moves on. The temp file is created in the note's own directory so the mv is same-filesystem and atomic, and a read-only note dir makes set_field fail cleanly (this is what the best-effort test exercises).

recall.sh — stamping

After the selection phase produces the three note sets (matched scoped, one-hop linked, global) and before/independent of CAP rendering, stamp each unique selected note exactly once (the existing inc_set dedup already enumerates them):

cur="$(field "$f" recall_count)"; cur="${cur:-0}"
case "$cur" in (*[!0-9]*) cur=0 ;; esac      # tolerate garbage
next=$(( cur + 1 ))
{ set_field "$f" recall_count "$next" \
    && set_field "$f" last_recalled "$(_woo_now)" ; } \
  || echo "recall: stamp failed $(basename "$f")" >&2

Stamping never changes stdout and never affects exit status — recall still exits 0. Output sections (Scoped / Linked / Global) are unchanged.

concurrency prefetch.sh:707 calls recall.sh exactly once per review, before any angle fan-out — so there is no parallel-stamp race within a review. Sequential reviews on the same checkout simply accumulate counts.

doctor.sh — dead-note check

Per note, after the existing field/scope/link checks:

upd="$(field "$f" updated)"
[ -z "$upd" ] && continue                    # no age basis → skip, no flag
rc="$(field "$f" recall_count)"; rc="${rc:-0}"
case "$rc" in (*[!0-9]*) rc=0 ;; esac
upd_e="$(_woo_epoch "$upd")" || continue     # unparseable date → skip
now_e="$(_woo_epoch "$(_woo_now)")"
age_days=$(( (now_e - upd_e) / 86400 ))
if [ "$age_days" -gt "${WOOSTACK_DEAD_DAYS:-90}" ] && [ "$rc" -eq 0 ]; then
  warn "$base: dead note — written ${age_days}d ago, never recalled (prune candidate)"
fi

Warning only (exit 0). A note with no updated:, or any recall_count > 0, or age within threshold is not flagged.

Known coverage limit (documented, not fixed here): the dead check requires updated:, which is optional. Notes without it are silently uncovered. The clean fix is for distillation (§7) to stamp updated: on every note it writes — tracked as a follow-up, not in this increment. doctor takes on no git/stat dependency. Also out of scope: flagging notes recalled long-ago-but-now-quiet (recall_count > 0, stale last_recalled) — that staleness/ranking work is seeded by last_recalled and deferred.

memory.md (contract)

  • §3 fields table: add recall_count and last_recalled as optional, tool-managed fields (written by recall.sh; never hand-edited).
  • §5 / §8: document the dead-note warning and the WOOSTACK_DEAD_DAYS (default 90) and WOOSTACK_NOW knobs.

6. Error handling

  • Stamp write fails (read-only FS, permission): set_field returns non-zero without overwriting the original; recall logs recall: stamp failed <note> to stderr and continues. Telemetry is best-effort; recall output and exit 0 are never compromised.
  • Garbage recall_count (hand-edited to non-numeric): treated as 0 in both recall (resets to 1) and doctor (treated as never-recalled). No crash under set -e.
  • Unparseable updated: date in doctor: _woo_epoch fails → that note is skipped for the dead check (no false flag, no error).
  • Missing updated:: skipped — there is no age basis, so no dead verdict.
  • Atomicity: temp+mv means a note is never left half-written even if the process dies mid-stamp.

7. Testing

All tests use the existing bash harness (assert.sh, mk_note) and inject WOOSTACK_NOW for determinism.

test-lib.sh — set_field

  • Update existing key leaves value changed, body + other fields intact.
  • Insert absent key adds it before the closing fence, body intact.
  • Value with spaces / a date round-trips through field().
  • _woo_now honors WOOSTACK_NOW; _woo_epoch parses an ISO date and is monotonic (later date → larger epoch).

test-recall.sh — stamping

  • After one run, a matched note has recall_count: 1 and last_recalled = injected date.
  • A second run bumps to recall_count: 2.
  • Linked (one-hop) and global (scope:*) notes are stamped too.
  • Existing recall output assertions (scoped/linked/global/cap/ordering) still pass — stamping does not alter stdout.
  • Best-effort: a read-only note dir makes recall log stamp failed to stderr yet still exit 0 with full output.

test-doctor.sh — dead note

  • Note with old updated: + no recall_count → warns "dead note", exit 0.
  • Same note with recall_count: 1 → not flagged.
  • Fresh updated: (within 90d) + recall_count 0 → not flagged.
  • No updated: field → not flagged.
  • WOOSTACK_DEAD_DAYS override changes the boundary.

8. Open questions

None blocking. Resolved during brainstorming + grilling: count semantics (cumulative events), write-back (unconditional best-effort), stamp scope (all selected), dead threshold (90d, env-overridable), _woo_epoch determinism (zero time-of-day), set_field safety on malformed notes, no concurrency race (single recall call per review).

Deferred by design, to be filed as follow-up issues at PR time:

  • Optional churn-guard for stamping during local reviews of unrelated branches — revisit only if real-world churn proves painful.
  • Distillation stamping updated: on every written note, so dead-note coverage is complete.
  • The "recalled-but-never-acted-on" signal (needs a downstream act signal that does not exist yet).
